
\section{Overview}

This chapter presents the performance and evaluation of the Medicare Bot system after full integration of hardware and software components. The system was tested under real operating conditions to verify its functionality, accuracy, and reliability.
The results presented in this chapter evaluate the performance of the system designed in Chapter \ref{systemarchitecture} and implemented in Chapter \ref{softwareimplementation}.
\section{System Functionality}

The system successfully performs the following operations:

\begin{itemize}
    \item Face detection and recognition of registered patients
    \item Verification of medication schedules
    \item Automated dispensing using servo motors
    \item Image capture during dispensing
    \item Logging of events to backend server
    \item Display of logs and images on frontend dashboard
\end{itemize}

All subsystems, including Raspberry Pi, Arduino, backend, and frontend, operate in coordination to achieve end-to-end functionality.

\section{Face Recognition Performance}

The face recognition system was tested under normal indoor lighting conditions.

\begin{itemize}
    \item Recognition accuracy: approximately 85\% -- 92\%
    \item Average processing speed: 4 -- 6 frames per second
    \item Recognition threshold: 0.45
\end{itemize}

The system performed reliably when faces were clearly visible and properly aligned. However, performance degraded under poor lighting or when faces were partially occluded.

\section{Dispensing Accuracy}

The dispensing mechanism was tested across multiple trials.

\begin{itemize}
    \item Successful dispensing rate: above 95\%
    \item Response time (recognition to dispense): approximately 2 -- 4 seconds
\end{itemize}

The system correctly dispensed medication when all conditions were satisfied, including identity verification and schedule matching.

\section{System Integration Performance}

The integration between different components was tested to evaluate data flow and communication reliability.

\subsection{Backend Communication}

The Raspberry Pi successfully transmitted logs and images to the backend server using HTTP requests. The use of multipart/form-data ensured reliable image uploads.

\subsection{Frontend Display}

The frontend dashboard successfully retrieved and displayed logs along with captured images. The auto-refresh mechanism ensured near real-time updates.

\section{Issues Encountered and Solutions}

Several issues were encountered during development and testing:

\subsection{Image Upload Error (422)}

Initially, log transmission failed with a 422 error due to incorrect request formatting. This was resolved by switching to multipart/form-data for sending both text and image data.

\subsection{Image Display Issue}

Images were not displayed on the frontend due to incorrect static file paths. This was fixed by correcting the URL from:

\begin{verbatim}
/api/static/images -> Incorrect
/static/images -> Correct
\end{verbatim}

\subsection{Power Supply Instability}

Servo motors caused instability when powered directly from the same source as the Raspberry Pi. This was resolved by using separate power supplies and a buck converter.

\subsection{Repeated Dispensing Issue}

The system initially dispensed medication multiple times due to improper handling of the last\_dispensed field. This issue highlighted the need for improved scheduling logic.

\section{System Limitations Observed During Testing}

The following limitations were observed during testing:

\begin{itemize}
    \item Reduced accuracy in low lighting conditions
    \item Slower processing speed due to Raspberry Pi limitations
    \item Sensitivity to multiple faces in the frame
\end{itemize}

\section{Overall Evaluation}

The Medicare Bot system successfully demonstrates a functional prototype of an AI-powered medication dispensing system. The integration of face recognition, embedded control, and web technologies enables automated and reliable operation.

Despite minor limitations, the system achieves its primary objective of safe and automated medication delivery, making it suitable for further development and real-world applications.
